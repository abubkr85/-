# تطبيق التحكم بإيماءات الرأس (Head Gesture Control)

مشروع مستقل بالكامل عن نسخة التحكم الصوتي — نفس الفكرة العامة (يتحكم
بالشاشة عبر أوامر: يمين، يسار، فوق، تحت، اضغط) لكن عبر **الكاميرا
الأمامية وتحليل إيماءات الرأس** بدل الميكروفون.

## الربط بين الإيماءة والأمر

| الإيماءة | الأمر |
|---|---|
| لف الرأس يمين | يمين |
| لف الرأس يسار | يسار |
| رفع الرأس | فوق |
| خفض الرأس | تحت |
| غمزة عين واحدة (يمين أو يسار) | اضغط |

**ملاحظة مهمة:** رمشة العينين معًا (كإغلاق طبيعي أثناء المشاهدة) **لا
تُحتسب** — فقط غمز عين واحدة لوحدها (وين الثانية تبقى مفتوحة بوضوح)
يُعتبر أمر "اضغط"، لتفادي التفعيل غير المقصود.

## التقنية المستخدمة

- **CameraX**: لتشغيل الكاميرا الأمامية بالخلفية باستمرار
- **ML Kit Face Detection (نسخة bundled/مضمّنة)**: لتحليل كل إطار
  واستخراج:
  - `headEulerAngleY` (زاوية اللف يمين/يسار)
  - `headEulerAngleX` (زاوية الرفع/الخفض)
  - `leftEyeOpenProbability` و`rightEyeOpenProbability` (لكشف الغمزة)

اخترت النسخة **المضمّنة (bundled)** من ML Kit عمدًا (مو نسخة Google
Play Services) لأنها تعمل أوفلاين بالكامل بدون اعتماد على خدمات جوجل —
هذا يفيد بالذات لو الجهاز (مثل بعض أجهزة شاومي) خدمات جوجل فيه منقوصة.
الكلفة: حجم APK أكبر شوي (~7 ميجا إضافية).

## لا يوجد معاينة كاميرا (Preview) على الشاشة
الخدمة تستخدم الكاميرا فقط للتحليل بالخلفية (`ImageAnalysis`)، ما
تعرض أي معاينة مرئية — يعني الكاميرا تشتغل لكن ما تشوف نفسك على
الشاشة. هذا مقصود لتوفير الموارد، بس ممكن نضيف معاينة اختيارية لاحقًا
لو احتجت تتأكد بصريًا إن الكشف شغّال صح.

## المعايرة (Calibration)
بما إن اتجاه دوران الكاميرا الأمامية ممكن يختلف حسب الجهاز، أضفت
مفتاحين بشاشة الإعدادات:
- **عكس يمين/يسار**
- **عكس فوق/تحت**

لو أول ما تجرب لقيت الاتجاهات معكوسة (تلف يمين ويستجيب كأنه يسار)،
فعّل المفتاح المناسب.

## خطوات البناء (نفس أسلوب المشروع السابق)
اتبع نفس خطوات رفع المشروع لـ GitHub وتشغيل GitHub Actions اللي
استخدمناها قبل. ملف `.github/workflows/build.yml` جاهز بنفس الإعدادات
المجربة والناجحة (Flutter 3.24.5، AGP 8.5.2، Kotlin 1.9.24، Gradle 8.9).

## `minSdk` مرفوع إلى 24
رفعت الحد الأدنى لنظام أندرويد من 26 إلى 24 لأن CameraX وML Kit
يحتاجونه، لكنه لسا يغطي الغالبية العظمى من الأجهزة الحديثة.

## نقاط يستحسن اختبارها
1. **إضاءة الغرفة**: كشف الوجه يحتاج إضاءة كافية، خصوصًا لكشف حالة
   العينين بدقة.
2. **زاوية إمساك الجوال**: امسك الجوال بحيث الكاميرا الأمامية تشوف
   وجهك بوضوح ومباشر قدر الإمكان.
3. **الحساسية**: لو الإيماءات تتفعل بسهولة زايدة أو ما تتفعل إلا بحركة
   كبيرة جدًا، عدّل شريطي الحساسية بشاشة الإعدادات.
4. **استهلاك البطارية**: تشغيل الكاميرا المستمر بالخلفية يستهلك بطارية
   أكثر من التعرف الصوتي بشكل ملحوظ — توقّع هذا الفرق.
